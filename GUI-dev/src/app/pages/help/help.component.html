Processes are, when they are bough, dedicated to one special organization. Users can create a organization and add a Users to its organization in the User Management section.

If Users have the Rule CREATE_APPROVAL assigned to their role they can see the Approval Page in the menu and approve processes. After approval, processes can be configured. In case a User has the rule CREATE_PURCHASE, new processes can be bought for the organisation.

User of an organization can configure a process for the execution. Therefore process roles are needed. A starter role is defined. User of the starter role can then start a process and assign the next tasks to a specific user of a role defined in the configuration.

Role and Rule configuration can be done in the User management section. Process can be created in the Create Process section. In case Processes are already started and assigned to a user, or users have to role to start a process, these processes can be found in the process execution section.

Processes can be bought for the organization in the process store and the dashboard gives an overview of processes of the organization. In the dashboard, available processes can be configured.
